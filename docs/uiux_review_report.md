# Stage 1 UIUX 휴리스틱 리뷰 보고서

> 리뷰 범위: `docs/ui_flow.md`, `docs/wireframes_stage1.md`

## 1) 총평
Stage 1은 **업로드 → 검증 → 분석 → 결과/수정** 흐름이 명확하고, 데스크톱/모바일 분리 설계와 상태(기본/경고/오류/로딩) 정의가 비교적 잘 되어 있습니다. 특히 진행 단계(1/3~3/3), 검증 체크리스트, 치수 수정 패널은 사용자 목표(빠른 분석 + 보정)에 직접 기여합니다.

다만 실제 사용성 관점에서 **오류 복구 가이드의 구체성**, **용어/CTA 일관성**, **모바일 입력 맥락 조작성**, **접근성(명도/포커스/보조기기)** 보완이 필요합니다. 현재는 개발 착수 가능한 초안이나, 운영 품질 기준에서는 핵심 개선이 선행되어야 합니다.

## 2) 휴리스틱 10항목 평가 (심각도 포함)

| # | 휴리스틱 | 평가 | 심각도 |
|---|---|---|---|
| 1 | 시스템 상태의 가시성 | 진행률(%)·남은시간·현재 단계 표시는 우수. 지연/실패 시 후속 행동 가이드 보강 필요 | Med |
| 2 | 시스템-현실 세계 매치 | 치수/mm/신뢰도 용어는 적절. 일부 기술 용어는 사용자 친화 문구 병기 필요 | Med |
| 3 | 사용자 통제와 자유 | 이전/취소/재업로드/원복 제공은 좋음. 취소 시 임시저장 정책 명확화 필요 | Med |
| 4 | 일관성과 표준 | CTA 명칭이 화면별 혼재. 경고/오류 카피 톤 통일 필요 | Med |
| 5 | 오류 예방 | 업로드 제약 제시는 좋음. 치수 입력 사전 예방형 가이드가 부족 | High |
| 6 | 기억보다 인식 | 리스트/체크리스트 구조로 인지 부담 낮음. 수정 전후 diff 부재 | Med |
| 7 | 유연성과 효율성 | 데스크톱 2컬럼 효율적. 반복 편집 단축 경로는 약함 | Low |
| 8 | 미니멀/심미적 설계 | 구조는 명확. 모바일 정보 밀집 시 우선순위 재정렬 필요 | Med |
| 9 | 오류 인지·진단·복구 | 오류 상태 정의는 있음. 원인-해결 중심 메시지 강화 필요 | High |
|10| 도움말/문서 | 도움말 진입점은 긍정적. 인라인 컨텍스트 도움 강화 필요 | Low |

## 3) 개선 권고 Top 10 (우선순위/기대효과)

1. **오류 메시지 처방형 템플릿 도입** (P1) — 실패 이탈률 감소
2. **치수 입력 실시간 검증(범위·단위·형식)** (P1) — 저장 단계 오류 감소
3. **CTA 네이밍 통일** (P1) — 의사결정 시간 단축
4. **분석 지연/실패 대안 경로 강화** (P1) — 작업 완료율 향상
5. **수정 전/후 값 비교(diff) 제공** (P2) — 신뢰성 향상
6. **신뢰도 임계치 기반 UX 분기** (P2) — 저품질 결과 저장 방지
7. **모바일 하단 CTA + 키보드 충돌 대응 규격화** (P2) — 모바일 완료율 개선
8. **기술 용어 사용자 친화 문구 병기** (P3) — 이해도 향상
9. **접근성 기준 명시(대비/포커스/터치영역)** (P3) — QA 리스크 감소
10. **단계별 인라인 마이크로카피 배치** (P3) — 학습 곡선 완화

## 4) 즉시 반영 가능한 Quick Wins 5개

1. Primary 버튼 라벨 통일
2. 치수 입력 placeholder 예시 + 단위(mm) 고정
3. 오류 문구를 “원인 + 해결 액션” 1줄 표준화
4. 결과 화면에 “최근 수정값” 배지 추가
5. 모바일 검증 항목을 접힘 카드로 단순화

## 5) 승인 판정

**판정: `Change Request`**

### 승인 전 필수 조건
1. 치수 입력 검증 UX 구체화(사전 예방 + 오류 복구)
2. 분석 실패/지연 복구 플로우 명문화
3. 핵심 CTA/상태 메시지 카피 일관성 정리

### 조건 충족 시
- Stage 1 UI 설계는 구현/QA 착수 가능한 수준으로 **Approve 가능**
