# TASK-002 산출물 — Stage1 사용자 플로우(이미지 업로드 중심)

## 1) IA (Information Architecture)

- **Stage1: 이미지 업로드**
  - 1-1. 업로드 랜딩
  - 1-2. 업로드 진행/검증
  - 1-3. AI 분석 중
  - 1-4. 분석 결과 요약
- **공통 레이어**
  - 전역 토스트(성공/경고/오류)
  - 도움말(촬영 가이드, 지원 포맷)
  - 재시도/다시 업로드 모달
- **다음 단계 연결**
  - Stage2 진입 CTA(디자인 개선 시작)

---

## 2) 주요 화면 4개

## 화면 A. 업로드 랜딩
**목적:** 무엇을 업로드해야 하는지 즉시 이해시키기  
**핵심 컴포넌트**
- 제목/설명: 정면 촬영, 충분한 조명, 텍스트 식별 가능
- 업로드 존(클릭 + 드래그앤드롭)
- 지원 형식/용량/권장 해상도 안내
- 샘플 이미지 링크
- CTA: `이미지 업로드`

---

## 화면 B. 업로드 진행/검증
**목적:** 업로드 상태와 품질 검사 결과를 투명하게 제공  
**핵심 컴포넌트**
- 진행률 바 + 상태 문구
- 파일 썸네일/파일명
- 검증 결과 배지
  - 통과
  - 경고(품질 저하 가능)
  - 실패(진행 불가)
- 액션: `재시도` / `다시 업로드` / (경고 시) `그대로 진행`

---

## 화면 C. AI 분석 중
**목적:** 대기 이탈 방지 및 신뢰 형성  
**핵심 컴포넌트**
- 단계형 로딩(영역 감지 → 텍스트 가독성 → 컬러 분석)
- 예상 소요 시간 안내
- 취소 후 이전 단계 복귀 옵션

---

## 화면 D. 분석 결과 요약
**목적:** Stage2 진입 동기 강화  
**핵심 컴포넌트**
- 분석 요약 카드(라벨 영역, 가독성, 컬러 특성)
- 개선 제안 2~3개(짧고 실행형)
- Primary CTA: `Stage2에서 디자인 개선하기`
- Secondary CTA: `이미지 다시 업로드`

---

## 3) 화면전이 (정상/예외)

## 정상 플로우
1. A(업로드 랜딩)  
2. B(업로드 진행/검증-통과)  
3. C(AI 분석 중)  
4. D(분석 결과 요약)  
5. Stage2 진입

## 예외 플로우
- **B에서 경고 발생**
  - `그대로 진행` → C
  - `다시 업로드` → A
- **B에서 실패 발생**
  - `재시도` → B
  - `다시 업로드` → A
- **C에서 네트워크 오류**
  - 자동 재시도(1~2회) → 성공 시 C 지속
  - 실패 시 오류 모달 → `재시도`(C) / `업로드로 이동`(A)
- **D에서 결과 신뢰 낮음**
  - `다시 업로드` 권장 노출 + A 복귀

---

## 4) 입력 최소화 원칙

- 텍스트 입력 없이 **파일 선택 1행동**으로 시작
- 파일 메타정보(형식/크기) 자동 인식, 사용자 재입력 금지
- 분석 옵션은 기본값 자동 적용(고급 옵션 숨김)
- 경고 상황도 이중 선택지만 제공(`그대로 진행`/`다시 업로드`)
- 모바일은 카메라/갤러리 선택을 OS 기본 시트로 위임
- 동일 파일 재업로드 시 이전 설정 자동 승계

---

## 5) 오류/빈상태 가이드

## 오류 가이드
- **원인 + 해결책 동시 제시**
  - 예: “용량이 10MB를 초과했어요. 이미지 크기를 줄여 다시 업로드해주세요.”
- 오류 메시지 규칙
  - 짧고 구체적(무엇이 문제인지)
  - 다음 행동 명확(어떻게 해결하는지)
  - 즉시 행동 버튼 포함(`재시도`, `다시 업로드`)
- 치명 오류(진행 불가)와 경고(진행 가능) 시각적으로 구분

## 빈상태 가이드
- **초기 빈상태(A)**
  - 안내 문구 + 샘플 이미지 + 업로드 CTA
- **결과 없음(D)**
  - “분석 결과를 생성하지 못했어요” + 재분석/재업로드 액션
- **히스토리 없음(선택 시)**
  - “아직 업로드한 이미지가 없어요” + 첫 업로드 유도

## 6) 컨펌 체크리스트

- [ ] 업로드 목적/방법이 3초 내 이해된다  
- [ ] 지원 포맷/용량/권장 해상도가 명확히 보인다  
- [ ] 업로드 진행률과 현재 단계가 실시간으로 보인다  
- [ ] 경고와 실패가 구분되고, 각각 다음 행동이 명확하다  
- [ ] 분석 중 이탈 방지용 상태 메시지가 제공된다  
- [ ] 결과 화면에서 Stage2 진입 이유가 분명하다  
- [ ] 모든 예외 케이스에 재시도/복귀 경로가 있다  
- [ ] 키보드 접근/스크린리더 공지 등 접근성 기준을 충족한다  
- [ ] 모바일에서 한 손 조작 기준 CTA 배치가 적절하다  
- [ ] 이벤트 로깅(업로드 성공/실패, 경고, Stage2 클릭)이 정의되었다
